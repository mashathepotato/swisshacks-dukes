# Context

Shared context for the SwissHacks "Dukes" team. Keep this folder up to date so
anyone (human or AI agent) can get oriented fast.

**AI agents start here → [`AGENTS.md`](AGENTS.md)** — a dense, single-read
orientation (repo map, domain glossary, personas, invariants, commands). The
files below are the deeper detail it links into.

## Files

- `AGENTS.md` — agent entry point: read-this-first orientation, condensed.
- `challenge.md` — the challenge prompt, rules, judging criteria, deadlines.
- `project.md` — what we're building: idea, scope, MVP, stretch goals.
- `architecture.md` — stack, components, data flow, key decisions.
- `team.md` — who's who and who owns what.
- `decisions.md` — running log of decisions and why we made them.
- `agents.dot` / `agents.png` — the conceptual multi-agent architecture diagram.
- `voice-digest.dot` / `voice-digest.png` — focused voice-capture + dynamic-digest flow.
- `values.md` — the Client DNA value-axes (vocabulary + news/portfolio mappings).

## Conventions

- One topic per file; link between files rather than duplicating.
- Convert relative dates ("tomorrow") to absolute ones.
- When something changes, update the file rather than appending stale notes.
- Keep `AGENTS.md` factual and tight — it's optimised for fast agent ingestion,
  not prose; push narrative/detail down into the topic files.
